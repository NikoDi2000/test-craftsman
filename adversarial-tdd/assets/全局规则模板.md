# 测试有效性规则

所有Agent在处理测试时都必须遵守。

## 黄金法则
测试必须通过公开接口验证**行为**，而非**实现细节**。

## 禁止事项
- [ ] 测试私有方法或内部函数
- [ ] Mock内部协作者（仅Mock外部边界）
- [ ] 无需实现就直接通过的测试
- [ ] 看到实现代码后才写的测试
- [ ] 单个Agent同时负责测试设计和实现
- [ ] 修改测试来适配实现（应修复实现）
- [ ] 测试只覆盖正常路径，没有痛苦场景
- [ ] 使用"测试替身"替代核心业务逻辑验证
- [ ] 水平切片（批量写测试再批量写实现）

## 强制事项
- [ ] 写测试代码前先输出风险分析文档
- [ ] 至少3条正常路径测试
- [ ] 至少5条痛苦场景测试（空值、边界、溢出、并发、依赖、状态、幂等、时序）
- [ ] 至少3条变异假设文档化
- [ ] 重构前必须通过测试审计员批准
- [ ] 确认红灯：实现前测试必须失败
- [ ] 确认绿灯：实现后测试必须全部通过
- [ ] 审计员执行全部6项通用变异检查 + 技术栈特定变异
- [ ] 技术栈特定痛苦场景已覆盖（Flutter/FastAPI/Unity）

## 技术栈特定规则

### Flutter/Dart
- [ ] Widget测试不直接Mock `BuildContext`，通过 ` tester.pumpWidget()` 提供真实环境
- [ ] 异步测试使用 `async`/`await` 或 `pumpAndSettle`，不依赖固定延时
- [ ] 动画测试验证 `AnimationStatus` 和控制器释放

### FastAPI/Python
- [ ] Pydantic验证错误返回的HTTP 422包含精确的字段路径
- [ ] 数据库测试使用真实事务，验证 `commit`/`rollback` 效果
- [ ] 异步测试不阻塞事件循环，验证 `async`/`await` 链完整性

### Unity/C#
- [ ] 单元测试提取纯逻辑到非 `MonoBehaviour` 类
- [ ] 集成测试验证 `MonoBehaviour` 生命周期顺序
- [ ] 协程测试验证停止条件，不依赖固定帧数
