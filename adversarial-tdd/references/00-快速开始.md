# 快速开始：5分钟上手对抗性TDD

## 前提

- 已安装 OpenCode
- 项目根目录已有 `.opencode/` 配置目录

## 步骤

### 1. 安装本Skill

将 `adversarial-tdd` 目录复制到：

```bash
cp -r adversarial-tdd .opencode/skills/
```

### 2. 创建Agent

将项目根目录 `agents/` 下的 Agent 配置复制到：

- `.opencode/agents/测试设计师.md`（源文件：`agents/测试设计师.md`）
- `.opencode/agents/实现者.md`（源文件：`agents/实现者.md`）
- `.opencode/agents/测试审计员.md`（源文件：`agents/测试审计员.md`）

### 3. 创建全局规则

复制 `assets/全局规则模板.md`，保存为：

`.opencode/rules/测试有效性规则.md`

### 4. 配置权限

在 `opencode.json` 中添加：

```json
{
  "agent": {
    "build": {
      "permission": {
        "task": {
          "测试设计师": "allow",
          "实现者": "allow",
          "测试审计员": "allow"
        }
      }
    }
  }
}
```

### 5. 重启OpenCode并测试

```bash
opencode
```

然后输入：

```
给我实现一个简单的用户登录功能
```

观察OpenCode是否自动加载ATDD skill并调度三个Agent。

## 故障排查

| 问题 | 解决 |
|------|------|
| Agent未触发 | 检查 `opencode.json` 中 `task` 权限是否允许 |
| 测试设计师看到了实现 | 检查其 `read` 权限是否限制在测试目录 |
| 测试直接通过 | 检查是否已有同名实现，或测试设计有误 |
| 审计员未执行 | 确认工作流中阶段4是否被调用 |
| 水平切片发生 | 确保是垂直切片：一条测试→一条实现→重复 |
