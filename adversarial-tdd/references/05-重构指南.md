```markdown
# 重构指南

TDD 周期结束后，寻找以下重构候选人：

## 1. 消除重复

提取函数/类，消除跨测试和实现中的重复逻辑。

## 2. 过长方法

拆分为私有辅助函数。**保持测试在公开接口上**，不测试私有辅助函数。

## 3. 浅模块 → 深模块

合并或深化模块。将复杂度隐藏在简单接口后。

参考 `references/04-测试哲学与原则.md` 中的"深模块"章节。

## 4. 特性嫉妒

将逻辑移动到数据所在的位置。

## 5. 原始类型偏执

引入值对象（Value Objects）替代原始类型的过度使用。

## 6. 现有代码问题

新代码揭示的旧代码问题——一并修复。

## 重构铁律

- **绝不在红灯时重构**。先达到绿灯。
- 每次重构后运行测试。
- 如果测试碎了，回退。
```
