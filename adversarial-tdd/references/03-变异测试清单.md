# 变异测试清单

测试套件的价值取决于它抓住的bug数量。用以下"思维变异"验证测试有效性。

## 通用变异检查

### 1. 硬编码返回
将实现体替换为返回一个匹配某测试用例的常量。
- 其他测试会失败吗？如果不会，你需要更多多样化输入。

### 2. 边界差一
将 `>` 改成 `>=`，`<` 改成 `<=`，`+1` 改成 `+0`。
- 精确边界测试会失败吗？如果不会，添加精确边界测试。

### 3. 缺失守卫
移除空值检查、空集合检查或类型验证。
- 无效输入测试会失败吗？如果不会，添加守卫测试。

### 4. 缺失副作用
注释掉数据库写入、文件写入或事件发射。
- 状态验证测试会失败吗？如果不会，添加副作用测试。

### 5. 错误异常
更改异常类型或错误码。
- 异常类型特定测试会失败吗？如果不会，收紧异常断言。

### 6. 状态绕过
允许非法状态转换或跳过验证。
- 状态机测试会失败吗？如果不会，添加序列/转换测试。

## Flutter/Dart 特定变异

### F1. Widget 硬编码构建
将 `build()` 方法返回固定的 Widget 树，忽略输入参数。
- 不同输入参数的测试会失败吗？

### F2. 异步 Future 未等待
将 `await` 移除，让异步操作"fire and forget"。
- 依赖异步结果的测试会失败吗？

### F3. setState 条件错误
将 `setState` 调用条件反转或移除。
- UI 状态更新测试会失败吗？

### F4. Stream 未关闭
移除 `StreamController.close()` 或 `Subscription.cancel()`。
- 资源泄漏测试会失败吗？

### F5. BuildContext 滥用
在异步操作后直接使用 `context`（可能已卸载）。
- 生命周期测试会失败吗？

## FastAPI/Python 特定变异

### P1. Pydantic 字段硬编码
在模型中硬编码默认值，忽略传入参数。
- 不同输入的测试会失败吗？

### P2. 依赖注入替换为 Mock
将真实依赖替换为返回固定值的 Mock。
- 集成测试会失败吗？（注意：单元测试允许Mock，但需有集成测试兜底）

### P3. 异常处理泛化
将特定异常捕获改为通用的 `except Exception` 并返回 500。
- HTTP 状态码精确断言测试会失败吗？

### P4. DB 事务未提交
移除 `commit()` 或 `flush()`，数据只在内存中。
- 数据库状态验证测试会失败吗？

### P5. Async 转 Sync
将 `async def` 改为 `def`，在异步框架中阻塞事件循环。
- 并发测试会失败吗？

## Unity/C# 特定变异

### U1. MonoBehaviour 生命周期跳过
将初始化逻辑从 `Awake`/`Start` 移到构造函数（在编辑器模式下无效）。
- 场景加载测试会失败吗？

### U2. 协程未停止
移除 `StopCoroutine` 或 `StopAllCoroutines`，对象销毁后协程继续访问已销毁对象。
- 对象生命周期测试会失败吗？

### U3. 物理计算简化
将 `Rigidbody` 受力计算替换为直接设置 `transform.position`。
- 物理一致性测试会失败吗？

### U4. 主线程假设
在后台线程中直接访问 Unity API（如 `GameObject.Find`）。
- 多线程测试会失败吗？

### U5. 资源加载硬编码
将动态加载替换为直接引用场景中的预制体（忽略路径参数）。
- 资源变体测试会失败吗？

## 审计清单

对每种变异，标记：
- [ ] 已抓住：至少一个测试失败
- [ ] 遗漏：无测试失败 → 批准前必须补测试

## 规则
如果有**任何**变异被标记为"遗漏"，测试套件**无效**。拒绝并退回测试设计师。
