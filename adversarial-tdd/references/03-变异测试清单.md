# 变异测试清单

测试套件的价值取决于它抓住的bug数量。用以下方法验证测试有效性。

## 第一部分：路径覆盖检查

在开始变异测试前，先确认代码路径是否被覆盖。

### 覆盖率要求

| 度量 | 最低阈值 | 说明 |
|------|---------|------|
| 行覆盖率 | ≥ 80% | 代码被执行了 |
| 分支覆盖率 | ≥ 80% | 每个 if/else 都走了 |

### 覆盖率工具

| 技术栈 | 命令 |
|--------|------|
| Python | `pytest --cov --cov-branch` |
| Dart | `flutter test --coverage` |
| C# | `dotnet test /p:CollectCoverage=true` |

### 未覆盖分支分析

对每个未覆盖的分支，判断：

| 判断 | 行动 |
|------|------|
| 防御性代码（不应到达） | 添加注释说明，可接受 |
| 合法路径 | **必须补充测试** |

> 深入研究：[research/layer-03-path-coverage.md](../../research/layer-03-path-coverage.md)

## 第二部分：自动化变异测试

### 变异得分要求

| 得分 | 判定 | 行动 |
|------|------|------|
| < 40% | 测试套件几乎无效 | 大幅补充测试 |
| 40-70% | 有改进空间 | 分析存活变异体 |
| ≥ 70% | 合格 | 进入手动变异检查 |
| ≥ 85% | 优秀 | 快速检查即可 |

### 工具

| 技术栈 | 工具 | 命令 |
|--------|------|------|
| Python | mutmut | `mutmut run` |
| Dart | 手动 | 对照第三部分逐项检查 |
| C# | Stryker.NET | `dotnet stryker` |

### MuTAP 反馈循环

如果变异得分 < 70%，将存活的变异体反馈给测试设计师：

```
以下变异未被当前测试杀死，请补充能捕获这些变异的测试：
1. [具体变异描述，如 "将 age >= 18 改为 age > 18"]
2. ...
```

测试设计师基于此补充测试后，重新运行变异测试，直到得分 ≥ 70%。

> 深入研究：[research/layer-04-mutation-testing.md](../../research/layer-04-mutation-testing.md)

## 第三部分：手动变异检查

以下变异用"思维变异"方式逐项验证。对每种变异，在代码中模拟该错误，确认至少一个测试失败。

### 通用变异

#### M1. 硬编码返回
将实现体替换为返回一个匹配某测试用例的常量。
- 其他测试会失败吗？如果不会，需要更多多样化输入。

#### M2. 边界差一
将 `>` 改成 `>=`，`<` 改成 `<=`，`+1` 改成 `+0`。
- 精确边界测试会失败吗？如果不会，添加精确边界测试。

#### M3. 缺失守卫
移除空值检查、空集合检查或类型验证。
- 无效输入测试会失败吗？如果不会，添加守卫测试。

#### M4. 缺失副作用
注释掉数据库写入、文件写入或事件发射。
- 状态验证测试会失败吗？如果不会，添加副作用测试。

#### M5. 错误异常
更改异常类型或错误码。
- 异常类型特定测试会失败吗？如果不会，收紧异常断言。

#### M6. 状态绕过
允许非法状态转换或跳过验证。
- 状态机测试会失败吗？如果不会，添加序列/转换测试。

### Flutter/Dart 特定变异

#### F1. Widget 硬编码构建
将 `build()` 方法返回固定的 Widget 树，忽略输入参数。

#### F2. 异步 Future 未等待
将 `await` 移除，让异步操作"fire and forget"。

#### F3. setState 条件错误
将 `setState` 调用条件反转或移除。

#### F4. Stream 未关闭
移除 `StreamController.close()` 或 `Subscription.cancel()`。

#### F5. BuildContext 滥用
在异步操作后直接使用 `context`（可能已卸载）。

### FastAPI/Python 特定变异

#### P1. Pydantic 字段硬编码
在模型中硬编码默认值，忽略传入参数。

#### P2. 依赖注入替换为 Mock
将真实依赖替换为返回固定值的 Mock。
> 注意：单元测试允许 Mock，但需有集成测试兜底。

#### P3. 异常处理泛化
将特定异常捕获改为通用的 `except Exception` 并返回 500。

#### P4. DB 事务未提交
移除 `commit()` 或 `flush()`，数据只在内存中。

#### P5. Async 转 Sync
将 `async def` 改为 `def`，在异步框架中阻塞事件循环。

### Unity/C# 特定变异

#### U1. MonoBehaviour 生命周期跳过
将初始化逻辑从 `Awake`/`Start` 移到构造函数。

#### U2. 协程未停止
移除 `StopCoroutine`，对象销毁后协程继续访问已销毁对象。

#### U3. 物理计算简化
将 `Rigidbody` 受力计算替换为直接设置 `transform.position`。

#### U4. 主线程假设
在后台线程中直接访问 Unity API（如 `GameObject.Find`）。

#### U5. 资源加载硬编码
将动态加载替换为直接引用场景中的预制体。

## 审计清单

- [ ] 行覆盖率 ≥ 80%，分支覆盖率 ≥ 80%
- [ ] 未覆盖分支已分析（防御性代码标注，合法路径已补测）
- [ ] 自动化变异得分 ≥ 70%（如有工具）
- [ ] M1-M6 全部通过
- [ ] 技术栈特定变异全部通过
- [ ] 存活变异体已反馈给测试设计师

**规则**：任一项未通过，测试套件**无效**。拒绝并退回测试设计师。
