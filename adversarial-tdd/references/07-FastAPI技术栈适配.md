# FastAPI/Python 技术栈适配

## 语言特定空值
- Python 使用 `None`，但 Pydantic 会强制类型检查
- 必须测试：Pydantic 模型字段缺失、类型错误（如传入字符串到 int 字段）、可选字段为 None
- 必须测试：FastAPI `Query`/`Path`/`Body` 参数的默认值与必填混淆

## 异步与并发
- FastAPI 原生支持 `async/await`，但数据库驱动可能同步
- 必须测试：`async def` 被同步调用（如在线程池中运行）、事件循环冲突
- 必须测试：并发请求下的数据库连接池耗尽、事务隔离级别错误
- 必须测试：`BackgroundTasks` 在请求返回后执行失败的情况

## Web 框架边界
- 必须测试：HTTP 状态码边界（200/201/204/400/401/403/422/500）
- 必须测试：请求体超大（`Content-Length` 限制）、multipart 上传空文件
- 必须测试：路由参数类型转换失败（如 `/items/abc` 路由期望 int）
- 必须测试：依赖注入（`Depends`）的循环依赖、缺失依赖

## 数据层边界
- 必须测试：SQLAlchemy 懒加载在序列化时触发 N+1 查询
- 必须测试：数据库事务回滚（`rollback()`）是否真正撤销写入
- 必须测试：唯一约束冲突（如重复邮箱注册）的 HTTP 422/409 返回
- 必须测试：软删除 vs 硬删除的行为差异

## Mock 策略
- 外部 HTTP：使用 `responses` 库或 `pytest-httpx`
- 数据库：使用 `pytest-postgresql` 或内存 SQLite（但需注意 SQLite 与 PostgreSQL 行为差异）
- 缓存/队列：使用内存版 Redis (`fakeredis`) 或 RabbitMQ 模拟
- **禁止**：在集成测试中 Mock 数据库（应使用测试数据库实例）
- **禁止**：Mock Pydantic 验证（应测试真实验证行为）

## 痛苦场景追加清单

| 编号 | 场景 | 预期行为 |
|------|------|----------|
| P1 | JWT 过期/伪造/签名错误 | 返回 401，不进入业务逻辑 |
| P2 | SQLAlchemy 对象在 session 关闭后访问 | 抛出异常或正确预加载 |
| P3 | 文件上传超过大小限制 | 返回 413，不保存文件 |
| P4 | 依赖注入的缓存服务失效 | 降级或返回 503 |
| P5 | Pydantic 嵌套模型验证失败 | 返回 422，错误路径精确到字段 |
| P6 | 数据库死锁/连接超时 | 重试或返回 500，不泄露敏感信息 |
| P7 | 异步生成器/流式响应中断 | 正确关闭连接，无资源泄漏 |

## 变异假设追加

- 假设P1：如果实现者将 Pydantic 验证改为手动 `if` 判断 → 边界输入测试会失败
- 假设P2：如果实现者移除 `await` 直接返回 coroutine → 异步测试会失败
- 假设P3：如果实现者将 `raise HTTPException` 改为 `return {"error": ...}` → HTTP 状态码测试会失败
- 假设P4：如果实现者忘记 `db.commit()` → 数据库状态验证测试会失败
- 假设P5：如果实现者将依赖注入改为全局单例 → 并发隔离测试会失败
