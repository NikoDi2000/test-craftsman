# 风险矩阵模板

写任何测试代码前**必须**使用此模板。填满所有章节。

## 第零步：可测试性评估

在开始分析之前，先回答三个问题。任一答案为"否"，**先重构引入 Seam，再继续**。

| 问题 | 判断标准 |
|------|---------|
| **可控性** | 每个外部依赖（数据库、HTTP、文件系统）都能被替换吗？ |
| **可观测性** | 代码的输出可以直接观察吗（返回值 > 异常 > 状态修改）？ |
| **确定性** | 运行 100 次，结果始终一致吗？ |

如果不可测试，使用以下最小重构（按推荐顺序）：
1. 构造函数参数注入（Object Seam）
2. 方法参数注入（Object Seam）
3. 提取纯计算为独立函数（Extract Method）

> 深入研究：[research/layer-01-testability.md](../../research/layer-01-testability.md)

## 第三步：场景设计

### 正常路径（至少3条）

基于等价类和边界值生成：

1. 典型值路径：[标准有效输入 → 预期输出]
2. 下边界路径：[最小值 → 预期输出]
3. 上边界路径：[最大值 → 预期输出]

### 痛苦场景（至少5条）

| 编号 | 类别 | 场景 | 预期行为 |
|------|------|------|----------|
| 1 | 空值/空集 | 空输入 / 空集合 / 零值 | 抛出/拒绝/返回空 |
| 2 | 边界差一 | min-1 / max+1 / 边界精确值 | 正确处理或拒绝 |
| 3 | 溢出 | 超出范围的值、过大集合 | 错误/拒绝 |
| 4 | 并发 | 同时调用、重入访问 | 无数据损坏 |
| 5 | 依赖 | 网络超时、数据库失败 | 优雅降级 |
| 6 | 状态 | 非法状态转换、错误调用顺序 | 拒绝/回滚 |
| 7 | 幂等 | 同一输入执行两次 | 相同输出，无重复副作用 |
| 8 | 时序 | 慢路径、超时、竞态 | 确定性结果 |

### 技术栈特定补充

**Flutter/Dart**：
- null safety 边界（非空类型接收 null）
- Widget Key 冲突
- BuildContext 跨异步边界有效性
- 屏幕旋转后状态恢复

**FastAPI/Python**：
- Pydantic 字段缺失/类型错误
- 依赖注入循环
- async 函数被同步调用
- DB 连接池耗尽

**Unity/C#**：
- null 组件访问
- MonoBehaviour 未启用时调用
- 协程在对象销毁后继续
- 帧率波动下的时序依赖

## 第五步：预言类型选择

对每个测试场景，标记预言类型：

| 场景 | 预言类型 | 说明 |
|------|---------|------|
| [正常路径] | 精确/属性/蜕变/部分 | [原因] |
| [边界场景] | 精确/属性/蜕变/部分 | [原因] |

**选择规则**：
- **确定性场景**（算法、数据变换）→ 精确预言
- **非确定性场景**（AI、随机、物理）→ 部分预言 + 蜕变预言
- **解析/验证场景** → 属性预言（结构、类型、范围）

**禁止**：对 AI 输出使用精确断言（`assert result == "..."`）
