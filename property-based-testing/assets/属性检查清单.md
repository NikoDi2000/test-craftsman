# 属性检查清单

在为目标代码编写 PBT 测试前，逐项勾选确认。

## 识别阶段

- [ ] 代码中有序列化/反序列化对？（encode/decode、toJSON/fromJSON）
- [ ] 代码中有验证器？（validate、is_valid、check）
- [ ] 代码中有数据变换？（sort、filter、map、normalize）
- [ ] 代码中有算法/纯函数？（数学运算、排序、加密）
- [ ] 代码中有自定义数据结构？（add/remove/get）
- [ ] 代码涉及 AI/LLM 输出？（PydanticAI、API 返回非确定性结果）

## 属性选择

至少选择 2 种属性模式：

- [ ] Roundtrip：encode→decode 不变
- [ ] Idempotence：f(f(x)) = f(x)
- [ ] Invariant：变换后某属性不变
- [ ] Commutativity：顺序不影响结果
- [ ] Associativity：分组不影响结果
- [ ] Identity：单位元不改变值
- [ ] Inverse：逆操作回到原点
- [ ] Oracle：对比参考实现
- [ ] Easy-to-verify：验证比计算简单
- [ ] No-crash：合法输入不崩溃

## 编写阶段

- [ ] 使用了正确的技术栈工具（Hypothesis/glados/FsCheck）
- [ ] 策略自动覆盖了边界（Hypothesis 自带边界生成，无需手动枚举）
- [ ] 包含了至少一个 `@example` 回归用例
- [ ] 测试名称描述了属性，而非具体输入

## 运行阶段

- [ ] 默认 100 次运行全部通过
- [ ] 如有失败，shrink 后的最小反例已记录
- [ ] 发现的反例已固定为 `@example`

## AI/LLM 输出额外检查

- [ ] 检查了 JSON 结构合法性
- [ ] 检查了必填字段存在
- [ ] 检查了字段类型正确
- [ ] 检查了值范围合理
- [ ] 检查了无 PII 泄露
- [ ] 检查了相同输入的结构一致性
