# AI/LLM 输出测试

## 为什么 PBT 是唯一有效方式

LLM 输出是非确定性的——相同的 prompt，两次调用可能得到不同的结果。传统测试 `assert result == expected` 完全失效。

PBT 不断言精确值，而断言**结构属性**——这些属性无论 AI 输出什么内容都应该成立。

## ⚠️ 成本警告

AI 调用有成本。Hypothesis 默认 100 个用例，每个用例可能调用 AI 1-5 次 → 单个测试可能产生 100-500 次 AI 调用。

**必须限制用例数**：

```python
from hypothesis import settings

@settings(max_examples=10)  # AI 测试最多 10 个用例
@given(st.text(min_size=1, max_size=200))
def test_ai_response_is_valid_json(prompt):
    ...
```

| 测试类型 | 建议 max_examples | 原因 |
|---------|------------------|------|
| 结构合法性 | 10 | 成本高 |
| 类型一致性 | 10 | 成本高 |
| 蜕变测试 | 5 | 每个用例调用 2+ 次 |
| PII 泄露 | 10 | 成本高 |
| 幂等性 | 5 | 每个用例调用 2 次 |

**CI 建议**：AI 输出测试放在单独的 test suite 中，用 marker 标记，按需运行而非每次 CI 都跑。

## 准备工作

```python
from pydantic_ai import Agent

agent = Agent('openai:gpt-4o')
```

所有示例假设 `agent` 已初始化。

## 核心模式

### 1. 结构合法性

```python
from hypothesis import given, strategies as st, settings
import json

@settings(max_examples=10)
@given(st.text(min_size=1, max_size=200))
def test_ai_response_is_valid_json(prompt):
    result = agent.run_sync(prompt)
    try:
        data = json.loads(result.data)
    except json.JSONDecodeError:
        assert False, f"Output is not valid JSON: {str(result.data)[:100]}"

@settings(max_examples=10)
@given(st.text(min_size=1, max_size=200))
def test_ai_response_has_required_fields(prompt):
    result = agent.run_sync(prompt)
    data = json.loads(result.data)
    assert "response" in data
    assert "confidence" not in data or isinstance(data["confidence"], (int, float))
```

### 2. 类型一致性

```python
@given(st.text(min_size=1))
def test_ai_response_types_consistent(prompt):
    result = agent.run_sync(prompt)
    data = json.loads(result.data)

    if "items" in data:
        assert isinstance(data["items"], list)

    if "count" in data:
        assert isinstance(data["count"], (int, float))
```

### 3. 值范围合理性

```python
@given(st.text(min_size=1))
def test_ai_response_values_in_range(prompt):
    result = agent.run_sync(prompt)
    data = json.loads(result.data)

    if "confidence" in data:
        assert 0.0 <= data["confidence"] <= 1.0

    if "items" in data and isinstance(data["items"], list):
        assert len(data["items"]) <= 100
```

### 4. PII 泄露检查

```python
import re

@given(st.text(min_size=1))
def test_ai_response_no_pii_leak(prompt):
    result = agent.run_sync(prompt)
    text = str(result.data)

    assert not re.search(r'[\w\.-]+@[\w\.-]+\.\w+', text)
    assert not re.search(r'\b\d{11}\b', text)
```

### 5. 幂等性（结构层面）

```python
@given(st.text(min_size=1, max_size=100))
def test_ai_response_structure_idempotent(prompt):
    result1 = json.loads(agent.run_sync(prompt).data)
    result2 = json.loads(agent.run_sync(prompt).data)

    assert set(result1.keys()) == set(result2.keys())

    for key in result1:
        assert type(result1[key]) == type(result2[key])
```

## PydanticAI 专用

PydanticAI 的 `result.data` 返回 Pydantic 模型，天然适合 PBT：

```python
from pydantic import BaseModel
from pydantic_ai import Agent

class BookRecommendation(BaseModel):
    title: str
    author: str
    reason: str

agent = Agent('openai:gpt-4o', result_type=BookRecommendation)

@given(st.text(min_size=1, max_size=200))
def test_book_recommendation_structure(prompt):
    result = agent.run_sync(prompt)
    data = result.data

    # Pydantic 已保证类型，但还需要检查内容
    assert len(data.title) > 0
    assert len(data.author) > 0
    assert len(data.reason) > 0

@given(st.lists(st.text(min_size=1), max_size=5))
def test_batch_recommendations(prompts):
    """批量请求不互相污染"""
    results = [agent.run_sync(p).data for p in prompts]
    # 每个结果都是独立有效的
    for r in results:
        assert isinstance(r, BookRecommendation)
```

## 测试策略建议

| 测试层次 | 内容 | 频率 |
|----------|------|------|
| 结构合法性 | JSON 格式、必填字段 | 每次 CI |
| 类型一致性 | 字段类型正确 | 每次 CI |
| 值范围 | confidence 在 0~1、列表长度合理 | 每次 CI |
| PII 泄露 | 不包含邮箱/手机号等 | 每次 CI |
| 蜕变测试 | 同义改写结构一致、更具体问题结果更精确 | 每次 CI |
| 幂等性 | 相同输入结构一致 | 每日/合并前 |
| 内容质量 | 语义正确性 | 人工 review |

## 蜕变测试（Metamorphic Testing）

AI 输出没有"正确值"，但你知道输入变化应该怎样影响输出。

### MR1：同义改写 → 结构一致（恒等关系）

```python
@given(st.text(min_size=1, max_size=200))
def test_synonym_prompt_same_structure(prompt):
    result1 = agent.run_sync(prompt)
    result2 = agent.run_sync(f"请回答以下问题：{prompt}")

    data1 = json.loads(result1.data)
    data2 = json.loads(result2.data)
    assert set(data1.keys()) == set(data2.keys())
```

### MR2：更具体的问题 → 不更少信息（单调关系）

```python
@given(st.text(min_size=1, max_size=100))
def test_specific_prompt_not_less_info(topic):
    general = agent.run_sync(f"介绍 {topic}")
    specific = agent.run_sync(f"详细介绍 {topic} 的核心概念和应用")

    # 更具体的问题不应该返回更短的回答
    assert len(specific.data) >= len(general.data) * 0.5
```

### MR3：多次运行 → 不完全相同（随机性检查）

```python
@given(st.text(min_size=1, max_size=100))
def test_multiple_runs_not_identical(prompt):
    results = [agent.run_sync(prompt).data for _ in range(3)]
    # AI 有随机性，3 次结果不应完全相同
    assert len(set(results)) > 1
```

### MR4：添加无关信息 → 不影响核心结果

```python
@given(st.text(min_size=1, max_size=100))
def test_noise_doesnt_change_structure(prompt):
    clean = json.loads(agent.run_sync(prompt).data)
    noisy = json.loads(agent.run_sync(f"{prompt} （请用JSON格式回答）").data)
    assert set(clean.keys()) == set(noisy.keys())
```

## 禁止模式

- 断言 AI 输出的精确文本内容
- 只测一个 prompt 就认为覆盖了
- 忘记检查 PII 泄露
- 对非确定性输出使用 `==` 断言
