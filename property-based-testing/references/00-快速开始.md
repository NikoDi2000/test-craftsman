# 快速开始：5分钟上手属性驱动测试

## 前提

- Python 3.8+ / Dart 3.0+ / .NET 6+
- 已有对应语言的测试框架（pytest / dart test / xUnit）

## 步骤（以 Python 为例，其他语言参考 `02-技术栈适配.md`）

### 1. 安装 Hypothesis

```bash
pip install hypothesis
```

### 2. 写第一个 PBT 测试

```python
# test_first_pbt.py
from hypothesis import given, strategies as st

def add(a, b):
    return a + b

@given(st.integers(), st.integers())
def test_addition_commutative(a, b):
    assert add(a, b) == add(b, a)

@given(st.integers(), st.integers(), st.integers())
def test_addition_associative(a, b, c):
    assert add(add(a, b), c) == add(a, add(b, c))

@given(st.integers())
def test_addition_identity(a):
    assert add(a, 0) == a
```

### 3. 运行

```bash
pytest test_first_pbt.py -v
```

Hypothesis 默认运行 100 次，每次用不同的随机整数。如果全部通过，输出类似：

```
test_first_pbt.py::test_addition_commutative PASSED
test_first_pbt.py::test_addition_associative PASSED
test_first_pbt.py::test_addition_identity PASSED
```

### 4. 故意写个 bug 试试

```python
def broken_sort(lst):
    return []  # 故意返回空列表

@given(st.lists(st.integers()))
def test_sort_preserves_length(lst):
    result = broken_sort(lst)
    assert len(result) == len(lst)  # 这个会失败
```

运行后会看到 Hypothesis 找到了反例并自动 shrink 到最小形式。

### 5. 在你的实际项目中用

从 `references/01-属性模式目录.md` 中找到匹配你代码的模式，用 `references/02-技术栈适配.md` 找到对应语言的工具。

## 故障排查

| 问题 | 解决 |
|------|------|
| 测试太慢 | `@settings(max_examples=50)` 减少运行次数 |
| 生成的数据不符合预期 | 用 `assume()` 过滤或自定义策略 |
| 找不到反例 | 增加 `max_examples` 或用 `@example` 加已知反例 |
| Flaky 测试 | 检查代码是否有随机性或外部依赖 |
| 想复现某次失败 | Hypothesis 会打印 seed，用 `@seed()` 复现 |
