# 属性模式目录

## 1. Roundtrip（往返）

encode → decode 必须回到原点。

```python
from hypothesis import given, strategies as st
import json

@given(st.integers(), st.text())
def test_json_roundtrip(age, name):
    user = {"name": name, "age": age}
    encoded = json.dumps(user)
    decoded = json.loads(encoded)
    assert decoded == user
```

**适用场景**：序列化/反序列化、编解码、网络传输、持久化。

**你的项目**：Pydantic `model_dump_json()` → `model_validate_json()`；Flutter JSON 序列化；Unity 资源序列化。

---

## 2. Idempotence（幂等）

多次执行 f，结果与执行一次相同。

```python
@given(st.lists(st.integers()))
def test_sort_idempotent(lst):
    once = sorted(lst)
    twice = sorted(once)
    assert once == twice
```

**适用场景**：排序、去重、归一化、清理函数、API PUT 操作。

---

## 3. Invariant（不变量）

变换前后，某些属性永远不变。

```python
@given(st.lists(st.integers()))
def test_sort_preserves_elements(lst):
    result = sorted(lst)
    assert set(result) == set(lst)       # 元素不丢
    assert len(result) == len(lst)       # 长度不变
    assert all(a <= b for a, b in zip(result, result[1:]))  # 升序
```

**适用场景**：所有数据变换操作。

**你的项目**：Flutter 列表过滤不丢数据；FastAPI 数据校验不改变有效字段。

---

## 4. Commutativity（交换律）

操作顺序不影响结果。

```python
@given(st.integers(), st.integers())
def test_addition_commutative(a, b):
    assert a + b == b + a
```

**适用场景**：数学运算、多个 filter 的先后顺序、merge 操作。

---

## 5. Associativity（结合律）

分组方式不影响结果。

```python
@given(st.integers(), st.integers(), st.integers())
def test_addition_associative(a, b, c):
    assert (a + b) + c == a + (b + c)
```

**适用场景**：数学运算、数据管道组合、字符串拼接。

---

## 6. Identity（单位元）

存在一个值使操作不变。

```python
@given(st.integers())
def test_addition_identity(a):
    assert a + 0 == a  # 0 是加法的单位元

@given(st.text())
def test_string_identity(s):
    assert s + "" == s  # "" 是字符串拼接的单位元
```

**适用场景**：加法（0）、乘法（1）、字符串拼接（""）、列表拼接（[]）。

---

## 7. Inverse（逆操作）

操作 + 逆操作 = 回到原点。

```python
@given(st.lists(st.integers()), st.integers())
def test_push_pop(lst, item):
    original = list(lst)
    lst_copy = list(lst)
    lst_copy.append(item)
    lst_copy.pop()
    assert lst_copy == original
```

**适用场景**：push/pop、add/remove、create/delete、undo/redo。

---

## 8. Oracle（参照物）

对比一个简单但正确的参考实现。

```python
def optimized_abs(x):
    return x if x >= 0 else -x

@given(st.integers())
def test_abs_vs_builtin(x):
    assert optimized_abs(x) == abs(x)  # 标准库作为 Oracle
```

**适用场景**：优化版 vs 暴力版、新实现 vs 旧实现、并行版 vs 串行版。最简单的情况是用标准库函数作为 Oracle。

---

## 9. Easy-to-verify（易验证）

验证结果比计算结果简单。

```python
def is_sorted(lst):
    return all(a <= b for a, b in zip(lst, lst[1:]))

@given(st.lists(st.integers()))
def test_sort_produces_sorted(lst):
    result = sorted(lst)
    assert is_sorted(result)  # 比验证"排序正确"简单得多
```

**适用场景**：排序（验证升序）、搜索（验证结果在集合中）、因数分解（验证乘积）。

---

## 10. No-crash（不崩溃）

合法输入下绝不崩溃。

```python
from hypothesis import given, strategies as st, settings

def safe_parse(s: str) -> int:
    return int(s) if s.isdigit() else 0

@given(st.text())
@settings(max_examples=500)
def test_parse_never_crashes(s):
    try:
        safe_parse(s)
    except ValueError:
        assert False, f"Unexpected crash on input: {s!r}"
```

**适用场景**：解析器、API 端点、任何接受外部输入的函数。关键在于"所有输入都不应该让程序崩溃"。

---

## 额外模式：Metamorphic（蜕变）

不知道正确答案，但知道输入变化后输出如何变化。

```python
@given(st.lists(st.integers()))
def test_sort_append_then_sort(lst):
    # 追加一个大值再排序 = 先排序再追加
    result1 = sorted(lst + [999])
    result2 = sorted(lst) + [999]
    assert result1 == result2
```

**适用场景**：AI/LLM 输出测试（不知道精确答案，但知道结构规则）、科学计算。
