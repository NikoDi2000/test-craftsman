# Agent 模式切换

同一个 Agent 在不同 Skill 下切换工作模式。Agent 定义文件中包含所有模式的指令，Skill 触发时指定使用哪种模式。

## @测试设计师 的三种模式

### ATDD 模式

| 维度 | 内容 |
|------|------|
| 触发 Skill | `adversarial-tdd` |
| 分析框架 | 风险矩阵（可测试性/接口契约/输入空间/场景/变异假设/预言） |
| 参考文档 | `adversarial-tdd/references/02-风险矩阵模板.md` |
| 输出 | 失败测试 + 风险文档 |
| 交接信号 | `TESTS_READY` |

风险分析模板：

```markdown
## [功能名] 风险分析

### 可测试性评估
- 可控性：外部依赖都能替换吗？
- 可观测性：输出能直接观察吗？
- 确定性：运行100次结果一致吗？

### 接口契约
- 前置条件：[参数约束、状态约束]
- 后置条件：[返回值、状态变更、副作用]
- 不变量：[操作前后永远为真的属性]

### 输入空间建模
- 等价类划分：[有效/无效等价类]
- 边界值：[min-1, min, min+1, typical, max-1, max, max+1]
- 参数组合：[交互风险]

### 场景设计
- 正常路径（≥3条）
- 痛苦场景（≥5条）

### 变异假设（≥3条）
"如果实现者犯了 [X] 错误，测试 [Y] 会失败"

### 预言选择
- 确定性场景 → 精确预言
- 非确定性场景 → 部分预言 + 蜕变预言
```

### API 集成模式

| 维度 | 内容 |
|------|------|
| 触发 Skill | `api-integration-testing` |
| 分析框架 | 六步法（可测试性/HTTP 语义/认证矩阵/安全清单/DB 验证） |
| 参考文档 | `api-integration-testing/SKILL.md` |
| 输出 | 测试场景清单 |
| 交接信号 | 场景清单 → @集成测试工程师 |

六步法检查清单：

1. 可测试性评估（可控性/可观测性/确定性）
2. HTTP 语义分析（方法安全性/幂等性、状态码覆盖）
3. 认证场景矩阵（6种 Token 场景）
4. 安全测试清单（BOLA、注入、数据暴露）
5. 数据库状态验证

### PBT 模式

| 维度 | 内容 |
|------|------|
| 触发 Skill | `property-based-testing` |
| 分析框架 | 三问题（不变量/可逆操作/等价关系） |
| 参考文档 | `property-based-testing/SKILL.md` |
| 输出 | 属性测试 |
| 交接信号 | 无（单 Agent 完成） |

三问题：

1. 有什么不变量？（排序后长度不变）
2. 有什么可逆操作？（编码再解码等于原值）
3. 有什么等价关系？（两种实现产生相同结果）

## @测试审计员 的两种模式

### ATDD 审计模式

| 维度 | 内容 |
|------|------|
| 触发 Skill | `adversarial-tdd` |
| 审计重点 | 变异测试、覆盖率、通用6项 |
| 参考文档 | `adversarial-tdd/references/03-变异测试清单.md` |
| 判定标准 | 变异得分 ≥ 70%，0 个缺口 |

通用审计6项：

1. 硬编码测试：假设实现者返回硬编码值，其他测试会失败吗？
2. 边界测试：检查精确边界值
3. 空值测试：验证空集合、空指针、零值
4. 副作用测试：验证副作用是否被检查
5. 错误路径测试：验证错误条件是否抛出预期异常
6. 状态机测试：验证非法状态转换是否被拒绝

### API 集成审计模式

| 维度 | 内容 |
|------|------|
| 触发 Skill | `api-integration-testing` |
| 审计重点 | HTTP 语义、认证、安全、隔离 |
| 参考文档 | `api-integration-testing/SKILL.md` 六步法 |
| 判定标准 | HTTP 语义全覆盖 + 安全场景全覆盖 + 隔离性通过 |

审查清单：

1. HTTP 语义覆盖：方法安全性/幂等性是否验证？状态码是否精确？
2. 认证场景覆盖：6种 Token 场景是否全部覆盖？BOLA 是否测试？
3. 安全测试覆盖：注入攻击、敏感数据暴露是否测试？
4. 数据库状态验证：关键场景是否验证了数据库状态？
5. 测试隔离性：fixture 是否 `scope="function"`？`dependency_overrides` 是否清理？
