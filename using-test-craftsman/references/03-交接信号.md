# 交接信号

Agent 之间通过标准化信号交接，确保信息传递清晰、无歧义。

## 信号表

| 信号 | 发送者 | 含义 | 接收者应做什么 |
|------|--------|------|--------------|
| `TESTS_READY` | 测试设计师 | 测试已写好并确认失败 | 实现者开始实现 |
| `RED_CONFIRMED` | 实现者 | 实现前测试确实失败 | —（记录在案） |
| `GREEN_CONFIRMED` | 实现者 | 实现后测试全部通过 | 审计员开始审计 |
| `TESTS_ALREADY_PASSING` | 实现者 | 测试无需实现就通过 | 主 Agent 决定：丢弃测试或重设计 |
| `INTEGRATION_TESTS_READY` | 集成测试工程师 | 集成测试已通过 | 审计员开始审计 |
| `SECURITY_FINDINGS` | 集成测试工程师 | 发现安全问题 | 主 Agent 评估严重性 |
| `APPROVE` | 测试审计员 | 测试有效，无缺口 | 推进下一条测试或完成 |
| `REJECT` | 测试审计员 | 发现缺口 | 反馈给测试设计师补充 |
| `ESCALATE_TO_AUDIT` | 测试设计师/工程师 | 需要审计 | 审计员介入 |

## 信号流转图

### ATDD 流转

```
@测试设计师                    @实现者                    @测试审计员
    │                            │                            │
    ├── TESTS_READY ──────────►  │                            │
    │                            ├── RED_CONFIRMED            │
    │                            ├── GREEN_CONFIRMED ───────► │
    │                            │                            ├── APPROVE
    │  ◄─────────────────────────┼────────────────────────── ─┤ REJECT
    │  (REJECT 时补充测试)        │                            │
```

### API 集成测试流转

```
@测试设计师                    @集成测试工程师              @测试审计员
    │                            │                            │
    ├── 场景清单 ──────────────► │                            │
    │                            ├── INTEGRATION_TESTS_READY ►│
    │                            ├── SECURITY_FINDINGS ──────►│
    │                            │                            ├── APPROVE
    │  ◄─────────────────────────┼────────────────────────── ─┤ REJECT
    │  (REJECT 时补充场景)        │                            │
```

## 异常处理

| 异常信号 | 场景 | 处理方式 |
|---------|------|---------|
| `TESTS_ALREADY_PASSING` | 测试无需实现就通过 | 主 Agent 决定：1) 丢弃测试重设计 2) 检查是否已有实现 |
| `SECURITY_FINDINGS` | 集成测试发现安全漏洞 | 主 Agent 评估严重性，决定是否立即修复 |
| `REJECT`（连续3次） | 审计员反复拒绝 | 主 Agent 介入，评估是否需要调整测试策略 |
