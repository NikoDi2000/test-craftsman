# 跨 Skill 协作

Skill 可以组合使用，形成更完整的测试覆盖。

## ATDD + API 集成测试（FastAPI 后端推荐）

```
用户: "实现用户注册 API"
         │
    ┌────┴────┐
    │  ATDD   │  单元层：测试设计师写单元测试 → 实现者实现 → 审计员审计
    │  先行   │  覆盖：业务逻辑、数据验证、错误处理
    └────┬────┘
         │ 单元测试通过后
    ┌────┴──────────┐
    │ API 集成测试   │  集成层：设计师分析端点 → 工程师写集成测试 → 审计员审计
    │    后行        │  覆盖：HTTP 链路、认证、安全、DB 状态
    └───────────────┘
```

### 为什么 ATDD 先行

1. **速度**：单元测试秒级反馈，集成测试需要启动服务
2. **定位**：单元测试失败 → 精确定位到函数；集成测试失败 → 需要排查链路
3. **成本**：先在单元层验证核心逻辑正确，再在集成层验证组装正确

### 覆盖分工

| 层 | ATDD 覆盖 | API 集成测试覆盖 |
|----|----------|-----------------|
| 业务逻辑 | ✓ 验证规则、状态转换 | — |
| 数据验证 | ✓ Pydantic 模型验证 | — |
| HTTP 语义 | — | ✓ 状态码、方法安全性 |
| 认证授权 | — | ✓ Token 场景、BOLA |
| 安全 | — | ✓ 注入、数据暴露 |
| 数据库 | — | ✓ 状态验证、事务回滚 |

### 实际案例

用户要求："实现用户注册 API"

**ATDD 阶段**（单元层）：
- 测试设计师写：`test_user_model_validates_email`、`test_password_hashing`、`test_registration_with_duplicate_email_raises`
- 实现者实现：User 模型、密码哈希、注册服务
- 审计员审计：变异测试确认测试有效

**API 集成测试阶段**（集成层）：
- 测试设计师分析：POST /users 的 HTTP 语义、认证场景、安全清单
- 工程师写：`test_post_users_returns_201`、`test_post_users_with_duplicate_email_returns_409`、`test_bola_user_cannot_access_other_profile`
- 审计员审计：HTTP 语义覆盖 + 安全覆盖

## ATDD + PBT（数据验证逻辑推荐）

```
用户: "实现数据验证逻辑"
         │
    ┌────┴────┐
    │  ATDD   │  已知场景：边界值、空值、格式错误
    │  覆盖   │  人工能想到的所有情况
    └────┬────┘
         │
    ┌────┴────┐
    │   PBT   │  未知场景：框架随机生成输入
    │  补充   │  发现人工遗漏的边界和反例
    └─────────┘
```

### 为什么互补

- ATDD 覆盖"我知道会出错的地方"——人工设计的边界值和异常场景
- PBT 覆盖"我不知道会出错的地方"——框架随机生成输入发现意外反例

### 实际案例

用户要求："实现邮箱验证逻辑"

**ATDD 阶段**（已知场景）：
- `test_valid_email_accepted`
- `test_email_without_at_sign_rejected`
- `test_email_with_spaces_rejected`
- `test_email_with_double_dots_rejected`

**PBT 阶段**（未知场景）：
- 属性：`for all strings s, parse_email(format_email(s)) == s`（往返不变量）
- 属性：`for all valid emails e, validate(e) == True`（正例不变量）
- Hypothesis 可能发现：超长本地部分、Unicode 字符、IP 地址格式邮箱

## 三 Skill 组合（完整后端质量保障）

```
用户: "为 FastAPI 后端建立完整测试体系"
         │
    ┌────┴────┐
    │  ATDD   │  单元层：核心逻辑
    └────┬────┘
         │
    ┌────┴────┐
    │   PBT   │  属性层：数据验证、序列化
    └────┬────┘
         │
    ┌────┴──────────┐
    │ API 集成测试   │  集成层：HTTP 链路、安全
    └───────────────┘
```

### 测试分层映射

| 测试层 | Skill | 覆盖范围 | 速度 | 置信度 |
|--------|-------|---------|------|--------|
| 单元 | ATDD | 函数/类 | 快 | 中 |
| 属性 | PBT | 不变量/边界 | 中 | 高 |
| 集成 | API | 端到端链路 | 慢 | 最高 |

这对应测试钻石模型：单元层和集成层厚，属性层补充。
