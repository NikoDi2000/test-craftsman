# 认证授权测试模式

## 认证架构概览

FastAPI 认证通常涉及以下组件：

```
请求 → 中间件 → 依赖注入（oauth2_scheme）→ verify_token → User → 路由
```

## JWT 测试模式

### 基础配置

```python
# conftest.py
from jose import jwt
from datetime import timedelta

SECRET_KEY = "test-secret-key"
ALGORITHM = "HS256"

def create_test_token(
    data: dict,
    expires_delta: timedelta = None
) -> str:
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + (expires_delta or timedelta(minutes=30))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

def decode_test_token(token: str) -> dict:
    return jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
```

### 认证场景矩阵

| 场景 | Token 状态 | 预期状态码 | 错误信息 |
|------|-----------|-----------|----------|
| 正常访问 | 有效 | 200/201 | - |
| 无 Token | 无 | 401 | "Not authenticated" |
| 空 Token | "" | 401 | "Not authenticated" |
| 格式错误 | "InvalidFormat" | 401 | "Token signature verification failed" |
| 过期 | exp < now | 401 | "Token has expired" |
| 伪造 | 错误签名 | 401 | "Token signature verification failed" |
| 用户不存在 | 有效但用户被删除 | 401/404 | - |

### 完整测试示例

```python
import pytest
from datetime import timedelta

class TestJWTAuthentication:
    def test_valid_token_accepted(self):
        token = create_test_token({"sub": "user123", "role": "user"})
        response = client.get(
            "/protected",
            headers={"Authorization": f"Bearer {token}"}
        )
        assert response.status_code == 200

    def test_missing_token_rejected(self):
        response = client.get("/protected")
        assert response.status_code == 401
        assert "Not authenticated" in response.json()["detail"]

    def test_expired_token_rejected(self):
        token = create_test_token(
            {"sub": "user123"},
            expires_delta=timedelta(seconds=-1)
        )
        response = client.get(
            "/protected",
            headers={"Authorization": f"Bearer {token}"}
        )
        assert response.status_code == 401
        assert "expired" in response.json()["detail"].lower()

    def test_invalid_signature_rejected(self):
        bad_token = jwt.encode(
            {"sub": "user123", "exp": datetime.now(timezone.utc) + timedelta(hours=1)},
            "wrong-secret",
            algorithm="HS256"
        )
        response = client.get(
            "/protected",
            headers={"Authorization": f"Bearer {bad_token}"}
        )
        assert response.status_code == 401
```

## OAuth2 Password Flow 测试

```python
class TestOAuth2PasswordFlow:
    def test_login_success(self, test_user):
        response = client.post(
            "/token",
            data={
                "username": test_user.email,
                "password": "testpassword"
            }

        assert response.status_code == 200
        data = response.json()

            "/token",

                "username": "user@example.com",

    def test_login_nonexistent_user(self):
        response = client.post(
            "/token",

                "username": "nonexistent@example.com",
                "password": "anypassword"
            }
        )
        assert response.status_code == 401
```

## 权限层级测试

### 角色定义

```python
# app/models.py
class UserRole(str, Enum):
    USER = "user"
    MODERATOR = "moderator"
    ADMIN = "admin"
```

### 权限矩阵

| 资源 | User | Moderator | Admin |
|------|------|-----------|-------|
| GET /users | 自己的 | 自己的 | 所有 |
| POST /users | ❌ | ❌ | ✅ |
| DELETE /users | ❌ | ❌ | ✅ |
| GET /admin | ❌ | ❌ | ✅ |

### 权限测试示例

```python
class TestRBAC:
    def test_user_cannot_access_admin_endpoint(self):
        user_token = create_test_token({"sub": "user1", "role": "user"})
        response = client.get(
            "/admin/users",
            headers={"Authorization": f"Bearer {user_token}"}
        )
        assert response.status_code == 403

    def test_admin_can_access_admin_endpoint(self):
        admin_token = create_test_token({"sub": "admin1", "role": "admin"})
        response = client.get(
            "/admin/users",
            headers={"Authorization": f"Bearer {admin_token}"}
        )
        assert response.status_code == 200

    def test_user_cannot_delete_other_user(self):
        user_token = create_test_token({"sub": "user1", "role": "user"})
        response = client.delete(
            "/users/999",
            headers={"Authorization": f"Bearer {user_token}"}
        )
        assert response.status_code == 403

    def test_user_can_delete_own_profile(self):
        user_token = create_test_token({"sub": "user1", "role": "user"})
        response = client.delete(
            "/users/1",  # user1's own ID
            headers={"Authorization": f"Bearer {user_token}"}
        )
        assert response.status_code == 204
```

## 认证依赖覆盖

### Mock 认证依赖

```python
@pytest.fixture
def mock_current_user():
    """Mock 当前用户"""
    async def override():
        return User(id=1, username="testuser", role="user")

    app.dependency_overrides[get_current_user] = override
    yield
    app.dependency_overrides.pop(get_current_user, None)

@pytest.fixture
def admin_user():
    """Mock 管理员用户"""
    async def override():
        return User(id=999, username="admin", role="admin")

    app.dependency_overrides[get_current_user] = override
    yield
    app.dependency_overrides.pop(get_current_user, None)
```

### 使用 Mock 认证

```python
class TestWithMockedAuth:
    def test_admin_panel_with_mock(self, admin_user):
        # admin_user fixture 已覆盖 get_current_user
        response = client.get("/admin/panel")
        assert response.status_code == 200

    def test_user_panel_with_mock(self, mock_current_user):
        response = client.get("/user/profile")
        assert response.status_code == 200
```

## 安全测试边界

| 攻击类型 | 测试场景 | 预期行为 |
|----------|----------|----------|
| Token 重放 | 使用已使用的 Token | 正常处理或幂等拒绝 |
| Token 篡改 | 修改 payload | 签名验证失败 |
| 权限提升 | user token 伪装 role=admin | 正确拒绝 |
| 资源枚举 | 暴力猜测 ID | 适当限流/错误消息 |

## 常见问题

### 1. 测试 Token 与生产 Token 混用

**问题**：测试中使用硬编码的 SECRET_KEY，但生产使用环境变量

**解决**：
```python
# app/config.py
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    secret_key: str
    algorithm: str = "HS256"

# 测试覆盖
@pytest.fixture(autouse=True)
def mock_settings():
    app.dependency_overrides[get_settings] = lambda: Settings(
        secret_key="test-secret",
        algorithm="HS256"
    )
```

### 2. OAuth2 scheme 难以 Mock

**问题**：`OAuth2PasswordBearer` 在依赖注入前就验证 token

**解决**：Mock `verify_token` 依赖，而非 OAuth2 scheme
